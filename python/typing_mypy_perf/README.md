# typing, mypy, 그리고 성능 측정 비교

- 작성자(author): 장아현
- 작성일자(date): 2026.01.26
- 변경내역(description): 타입 힌트를 사용한 함수와 사용하지 않은 함수의 성능 비교

## 요구사항

- 두 가지 버전의 함수를 작성한다 ([함수] 입력 : 정수 리스트 → 출력 : 각 원소의 제곱을 더한 합)
  - `A 버전` : 타입 힌트를 사용하지 않은 함수
  - `B 버전` : 타입 힌트를 적용한 함수
- mypy를 이용해 버전 B의 타입을 검사해보고, 결과를 확인
- timeit을 사용하여 두 버전의 실행 성능을 각각 측정하고 성능 차이를 비교

## 파일 설명

- **sum_squares.py**
  - 입력 리스트의 각 원소의 제곱을 더한 합을 구하는 함수
    - 타입 힌트를 **사용하지 않은** `함수 A`
    - 타입 힌트를 **적용한** `함수 B`
      ```
      from typing import List
      ```
- **benchmark.py**
  - A, B 함수의 성능을 timeit을 활용해 실행 성능을 측정

## 결과 해석

|    항목     |      타입 힌트 없음 (A 버전)      |  타입 힌트 있음 (B 버전)   |
| :---------: | :-------------------------------: | :------------------------: |
|  정적 검사  |               불가                |        가능 (mypy)         |
|    성능     |             거의 동일             |         거의 동일          |
|   가독성    |               낮음                |            높음            |
| 코드 안전성 | 낮음<br />(런타임 오류 발생 가능) | 높음<br />(사전 검증 가능) |

## mypy 검사 결과

typing_square 함수(B 버전)에 대해 mypy 정적 타입 검사를 수행한 결과, 타입 오류 없이 정상 통과함을 확인하였다.

## 배운 점

타입 힌트는 런타임 성능 향상을 위한 기능이 아니라, 정적 분석을 통한 코드 안정성과 가독성 향상을 목적으로 사용됨을 실험을 통해 확인하였다.
